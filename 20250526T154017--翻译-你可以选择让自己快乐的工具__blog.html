你可以选择让自己快乐的工具

2025 年 5 月 20 日

在 Hacker News 和 Lobsters 上，我经常看到类似这样的博客标题：

  • 为什么我选择 Common Lisp 和 DragonflyBSD 来构建创业项目
  • 用 APL 重写 PyTorch（第六年进度更新）
  • 我永远、永远、永远都不会学 Docker

这类论调通常表现为：为何冷门事物优于流行事物。其理由总伪装成理性技术分析，实则永远都是诡辩。为什么？

因为人们做技术决策时，部分出于情感因素。他们选择某项技术，可能因为它让人感觉良好、熟悉，或是他们了解它。有人选择冷门技术作为交感巫术，就像用 ThinkPad 装 NetBSD 来 cos 威廉·吉布森笔下的主角。有人选择过时的语言，如 Lisp 或 Smalltalk，只因怀念施乐 PARC 的黄金时代，渴望与那段传统产生联结。人们寻找气质相投的工具：Ada 散发着「缓慢、保守、巴洛克」的气息，而 Rust 则标榜「快节奏、未经考验、暴发户」风格。有人用 Emacs，只因读过尼尔·斯蒂芬森那篇文章，觉得 VS Code 是给庸人用的，而 Emacs 才是诺斯替式的神器。

但许多人不敢承认这点！因为这违背他们的自我认知——他们自认为是毫无感情的笛卡尔式理性机器。于是他们编造合理化说辞。这类文章读多了，你就能识破套路。

为「冷门事物」辩护的论调往往淡化其缺点（「没错，我不得不花半年时间绕道为 Fortran 2023 实现 HTTP 服务器」），还编造些似是而非的优点。我曾见人鼓吹 Common Lisp 的伟大之处在于它有垃圾回收机制，仿佛作者患了某种认知障碍症，大脑识别不出 Python 的存在。

反对「流行事物」的论据则要么含糊其辞（「Docker 太复杂」），要么诉诸道德绑架（「那个社区很 toxic」），要么扯上身份认同（「Rust 让人软弱，C++才能保持警觉」）。有时这些批评确实成立，但放在更理性的天平上根本无足轻重。

不如快刀斩乱麻。

Emacs 就是个诺斯替教派。但那又如何？其实这样很棒。它能让你快乐，夫复何求？只要开心，你完全有权使用那些怪异、冷门、不便、过时甚至本该淘汰的东西。反正人终有一死。幸运的话能活到三十亿秒就该知足。听从内心的召唤吧，把 ZFS 塞进空气炸锅，用 Fortran 写报税代码。

我们使用工具来彰显其美德。你用 Tails 是因为它很赛博朋克？太棒了老兄。那就彻底投入吧。买件皮夹克。如果是为了美学，那就做到极致。让你的生活成为活生生的艺术品。去曼谷背包旅行，用 Gemini 写小说，用 2003 年的数码相机为你的 LiveJournal 拍照。把家庭群聊搬到 Signal 上。用 ISDN 公用电话接入站会，告诉项目经理联邦调查局在追捕你。然后写篇博客记录这一切。

但别糊弄我。别直视我的眼睛说 SNOBOL 是未来语言。别告诉老板你是经过理性成本效益分析才用 Prolog 重写前端的。

最重要的是，别欺骗自己。审视你的动机。如果纯粹出于执念追求某些事物，而忽视理性，某天醒来你可能会发现自己多年默默耕耘的竟是条死胡同。
