用caddy为服务器部署的服务添加https

前略，在自己的服务器上部署了Readeck服务。但是直接使用端口和IP的情况下，http流量有可能会被劫持，导向一些奇怪的网站。为了提高访问性，如果有一个域名，则可以利用子域名为Readeck服务添加https。

假设我们的域名是 jysp.jp ，则首先在服务器管理页面添加一个子域名解析，这里假设是 readeck.yjsp.jp 。然后我们安装Caddy，具体流程详见官方指南。

安装完成后，我们编辑 /etc/caddy/Caddyfile ，将内容替换如下：

readeck.yjsp.jp {
reverse_proxy 127.0.0.1:8000
}

再 caddy run 或者 systemctl start caddy ，就能自动申请https证书，并将访问 readeck.yjsp.jp 的流量转发至服务器上运行的readeck服务了。

对了，这里要记得在readeck配置文件里将ip改为 127.0.0.1 。
